Controller Functionality

Digital Prototyping

The SPUD controller was initially designed as a modular system consisting of two components: a handheld unit responsible for input controls and a wearable unit for tracking head or arm movement in 3D space.

The handheld controller integrates four buttons, a joystick, and an accelerometer for directional control and in-game actions.

The wearable controller features only a microcontroller and an accelerometer for motion tracking.

Controller Components

The SPUD controller consists of several carefully designed components, each serving a specific purpose in the overall functionality:

Finger Button Pins

These are multiple different sizes of pins that are used to press the physical button on the inside of the controller.

Menu Button

Ergonomically positioned menu button with a raised profile for easy access to game menus and system functions without compromising grip position.

Face Button

Custom designed buttons that provide tactile feedback for fingers with a familiar shape and feel.

Button Enclosure

Housing for the buttons used in the controller, universal for almost every button on the controller. Used to help guide the button pins into the correct position on the button.

Bumper Pin

Specialized pin design for bumper controls, needed something that would fit into a really small space. So the pin for the bumper was designed almost as a plunger.
